Uvod u strukture podataka i algoritme

We need to organise data and memory in order to make it easy to filter and search through it. Therefore organising a dictionary is completely different from organising MAP data(i.p geolocations) or financial data.

Structures

Discussed in two ways:
-logical/mathematical model: Abstract data types(no implementation details) also called ADT. Implementation: Discusses techniques on how to implement an ADT, because a single ADT can have multiple implementations. We will learn the cost of every implementation type.

LIST:

List is a collection of objects of same type. Functions:

  • Store a number of elements of the same type.
  • Write at a posotion
  • Modify at position
  • Read at a position
  • Insert
  • Remove

Example of list is an array.
Array implementation:

  • Define array size
  • Insert at x position (Insertion may cause shifting all elements in the array)
  • If array gets larger than the predefined size, create new with old array size x2 (double the size of the original one). Copy old array into new one. Delete old.

Linked list implementation:
-as opposing to arrays, linked lists contain a list of elements where every element points to the next.

Double linked lists
